# 项目内容目录

## 目录用途

本目录专门用于存放后续新增的项目主体内容，包括但不限于：

- **新增 Skill 模块**：新的 PPT 生成、设计分析等技能包
- **业务逻辑代码**：与 PPT 生成相关的核心业务逻辑
- **数据处理脚本**：数据预处理、后处理等辅助脚本
- **扩展功能**：项目未来迭代中新增的功能模块

## 存放规范

### 目录结构建议

```
project_content/
├── skills/               # 新增的 Skill 模块
│   └── <skill-name>/
│       ├── scripts/      # 脚本文件
│       ├── references/   # 参考文档
│       └── assets/       # 资源文件
├── utils/                # 通用工具函数
├── data/                 # 数据文件
└── tests/                # 测试文件
```

### 命名规范

- **文件夹命名**：使用小写字母和连字符（kebab-case），如 `ppt-generator`
- **Python 文件**：使用小写字母和下划线（snake_case），如 `generate_pptx.py`
- **资源文件**：保持原始命名，避免使用中文和特殊字符

### 文件规范

- 每个新增模块应包含 `README.md` 说明文件
- Python 脚本需包含文件头注释（功能说明、依赖、使用方法）
- 敏感信息（API Key、密码等）严禁提交到版本控制

## 注意事项

1. **依赖管理**：新增模块如有 Python 依赖，请同步更新根目录下的 `requirements.txt`
2. **代码风格**：遵循 PEP 8 编码规范
3. **版本兼容**：确保新增代码与项目现有 Python 版本和依赖库兼容
4. **路径引用**：使用相对路径时，以项目根目录为基准
5. **日志输出**：使用 `print` 输出到 `stderr` 进行日志记录，`stdout` 只输出最终结果
